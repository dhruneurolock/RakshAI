# RakshAI 30-Second Marketing Video Prompt

## Goal
Create a fast-paced 30-second marketing video that explains the project value, the end-to-end analysis flow, and how the agents work together inside RakshAI.

## Core Message
RakshAI is an automated web application penetration testing platform with deterministic, rule-based attack planning. The video should communicate that the system discovers, analyzes, executes, validates, reproduces proof-of-concept results, and supports remediation through a coordinated 7-agent workflow.

## Audience
- Security engineers
- DevSecOps teams
- AppSec managers
- Technical founders

## Style Direction
- Modern cyber-security aesthetic
- High-contrast UI screens, dark background, neon cyan and red accents
- Clean motion graphics, animated data flow, glowing nodes, and dashboard overlays
- Smooth transitions, no cheesy stock footage
- Confident, premium, technical, and slightly cinematic

## Video Length
30 seconds

## Format
- Aspect ratio: 16:9
- Resolution: 1080p or 4K
- Pace: energetic, polished, product-explainer style

## Scene-by-Scene Prompt

### 0-3s: Hook
Open with a dramatic digital security interface. Show a web app being scanned with moving lines, endpoint cards, and threat indicators. The screen should feel alive, like an intelligent security system is waking up.

On-screen text:
"Find vulnerabilities faster."

Voiceover:
"What if web application testing could think, analyze, and act automatically?"

### 3-8s: Project Overview
Show the RakshAI dashboard, then animate a central orchestrator node connecting to multiple modules. Display the project as a smart security platform, not just a scanner.

On-screen text:
"RakshAI"
"Automated Web App Penetration Testing"

Voiceover:
"RakshAI is an automated penetration testing platform built around deterministic rule-based attack planning."

### 8-14s: Analysis Flow
Visualize the project flow step by step with glowing arrows and connected blocks:
User input -> Orchestrator -> Scope Validation -> Discovery -> Attack Surface Map -> Rule Engine.

Use animated cards, endpoint lists, and architecture-style overlays to show analysis moving from input to insight.

On-screen text:
"Analyze"
"Map"
"Prioritize"

Voiceover:
"It analyzes scope, discovers targets, maps the attack surface, and routes everything through a structured rule engine."

### 14-21s: Agent Working
Show seven specialized agents working in parallel with different visual panels. Each agent should appear as an intelligent module processing tasks at the same time, with a central coordinator controlling the flow.

Agents to visualize:
- Coordinator Agent (Control)
- Recon Agent (Discovery)
- Attack Strategy Agent
- Exploit Agent (Execution)
- Validation Agent (Assurance)
- PoC Generator Agent (Reproducibility)
- Remediation Agent

Show each agent receiving data, executing tasks, and passing results forward.

On-screen text:
"Multi-Agent Execution"

Voiceover:
"A central coordinator controls seven specialized agents for discovery, strategy, execution, assurance, reproducibility, and remediation in a coordinated workflow."

### 21-26s: Intelligence and Safety
Show the rule engine as the brain of the system. Animate YAML rules, test case selection, payload binding, safety checks, and validation layers. Make the flow feel deterministic, controlled, and explainable.

On-screen text:
"Deterministic"
"Explainable"
"Safe"

Voiceover:
"Every decision is explainable, rule-driven, and safety-aware, so results are repeatable and trusted."

### 26-30s: Final Result
End on a polished report screen with clear findings, severity markers, and a clean dashboard summary. Finish with the logo or product name on a dramatic hero frame.

On-screen text:
"From discovery to report in one flow."
"RakshAI"

Voiceover:
"From discovery to remediation, RakshAI turns complex security testing into one intelligent flow."
